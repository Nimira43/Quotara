# Writing Clean Code

Clean code is simple:  
Write code your future self won’t want to fight in a car park.

It’s readable.  
It’s understandable.  
It doesn’t require a séance to figure out what `x2_final_v3_REAL.js` does.

## Principles

- **DRY:** Don’t Repeat Yourself (unless you enjoy debugging the same bug twice).  
- **YAGNI:** You Aren’t Gonna Need It (but you added it anyway, didn’t you).  
- **SOLID:** A set of rules everyone quotes and nobody fully remembers.

## Writing for the Future

Clean code is empathy.  
It’s kindness.  
It’s making sure the next developer doesn’t open your file and whisper,  
“What fresh hell is this?”

Use linters.  
Use formatters.  
Use your brain.
